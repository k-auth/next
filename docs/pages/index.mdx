# K-Auth

**설정 한 줄, 버튼 한 줄, 에러는 한국어로.**

Next.js에서 카카오, 네이버, 구글, 애플 로그인을 가장 쉽게 구현하세요.

## 왜 K-Auth를 만들었나요?

next-auth에 카카오/네이버 Provider가 있긴 해요. 그런데 직접 써보셨다면 아실 거예요:

- **네이버 로그인이 갑자기 안 돼요** - `expires_in` 타입 에러... 네이버가 OAuth 스펙을 안 지켜서 생기는 문제예요
- **scope 설정이 복잡해요** - 전화번호 받으려면 `phone_number`인지 `account/phone`인지 문서 뒤져야 해요
- **버튼을 또 만들어야 해요** - 카카오 노란색이 `#FEE500`인지 `#FFEB00`인지, 가이드라인 찾느라 시간 다 가요
- **에러가 영어예요** - `Invalid redirect_uri`라는데, 뭘 어떻게 고치라는 건지...

K-Auth는 이런 삽질을 대신 해뒀어요.

## 이렇게 달라져요

### 설정이 간단해요

```typescript
// 그냥 이게 끝이에요
export const { handlers, auth, signIn, signOut } = KAuth({
  kakao: {
    clientId: process.env.KAKAO_CLIENT_ID!,
    clientSecret: process.env.KAKAO_CLIENT_SECRET!,
    collect: { email: true, phone: true },  // scope? 몰라도 돼요
  },
});
```

### 버튼은 이미 있어요

각 서비스 [디자인 가이드라인](https://developers.kakao.com/docs/latest/ko/kakaologin/design-guide) 100% 준수. 색상, 아이콘, 텍스트 다 맞춰뒀어요.

```tsx
import { Button } from '@k-auth/next/ui';

<Button.Kakao onClick={() => signIn('kakao')} />  // 끝
```

### 에러는 한국어로

```
[K-Auth 오류] KAKAO_INVALID_REDIRECT_URI

메시지: Redirect URI가 등록되지 않았습니다.
힌트: 카카오 개발자센터 > 카카오 로그인 > Redirect URI에
      http://localhost:3000/api/auth/callback/kakao 를 추가하세요.
```

### next-auth 그대로 써요

K-Auth는 next-auth를 래핑했을 뿐이에요. GitHub, Discord 같은 다른 Provider도 함께 쓸 수 있어요.

```typescript
import GitHub from 'next-auth/providers/github';

KAuth({
  kakao: { ... },
  nextAuthConfig: {
    providers: [GitHub({ ... })],
    callbacks: { ... },
  },
});
```
